## 库基本使用方式

* 源码库规范

	1、不要提交源码编译后的二进制成果物到`svn`或`gitlab`！

    2、`.bak` `Thumbs.db` `output` `.idea` `.svn`等不要上传`svn`或`gitlab`！
  
	3、nodejs的引用包，如`node_moudle`目录下的包不要上传`svn`或`gitlab`！
	
* 成果物规范

	1、版本号规范:http://rdknown.hikvision.com/sc/doc/standard/view?id=50146b9b-221c-4b36-91e4-766eda605896&classname=com.hikvision.standard

	2、成果物命名规范：http://rdknown.hikvision.com/sc/doc/standard/view?id=f7732f8e-6ce5-41f9-8df0-9598b2429672&classname=com.hikvision.standard
	
* [日志提交规范]

    `格式`
    ```sh
    <type>(<scope>): <subject>
    // 空一行
    <body>
    // 空一行
    <footer>
    ```
    `type 类别`
    ```sh
    feature：新功能（feature），有需求单的填单号
    fix：修补bug，有缺陷单的填缺陷单号
    docs：文档（documentation）
    style：格式（不影响代码运行的改动）
    refactor：重构（即不是新增功能，也不是修改bug的代码变动）
    test：增加测试
    chore：构建过程或辅助工具的变动
    ```
    `样例`
    ```sh
    fix: minor typos in code

    see the issue for details on the typos fixed

    fixes issue #12 或者 缺陷单
    ```
